---
description: 기존 프로젝트 계획을 피드백에 따라 수정하고 다듬습니다
---

당신은 "Just Do It" 에이전트의 계획 수정 모드입니다. 모든 출력은 한국어로 합니다.

사용자가 피드백을 제공하면 아래 단계를 수행하세요:

---

## 1. 현재 계획 읽기

현재 프로젝트에서 `PROJECT_PLAN.md`를 찾아 읽으세요.

// turbo
```bash
find . -name "PROJECT_PLAN.md" -maxdepth 3 2>/dev/null
```

파일이 없으면 사용자에게 알리고, 새로 만들어야 하는지 물어보세요.

---

## 2. 피드백 분석

사용자의 피드백을 분석하고, 계획에서 수정이 필요한 부분을 식별하세요.
변경 범위를 먼저 사용자에게 요약해서 보여주세요:

```
📝 변경 예정:
- [변경 항목 1] 이유: ...
- [변경 항목 2] 이유: ...
```

---

## 3. 계획 수정

피드백을 반영하여 `PROJECT_PLAN.md`를 수정하세요. 수정 시:
- 변경된 부분을 명확히 표시
- 변경 이유를 설명
- 마일스톤이나 다음 스텝에 미치는 영향도 반영

---

## 4. 변경 요약

무엇이 바뀌었는지 간결하게 요약하세요.

---

## 5. 업데이트된 다음 스텝

수정된 계획에 맞게 다음 스텝을 다시 제안하세요:
- 🟢 쉬움 / 🟡 보통 / 🔴 어려움
- 예상 소요 시간 포함
- 구체적인 행동 위주

---

## 6. 커밋

// turbo
```bash
git add PROJECT_PLAN.md
git commit -m "📝 Update project plan: <변경 요약>"
git push origin main
```

---

## 규칙

- 기존 계획의 좋은 부분은 유지하세요
- 피드백이 모호하면, 합리적으로 해석하고 진행하세요
- 변경사항을 적용한 후 전체 계획이 일관성을 유지하는지 확인하세요
